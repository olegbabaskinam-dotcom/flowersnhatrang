### МЕТОД ОЦИФРОВКИ существующих месяцев (Фев и далее) — ВАЖНО
Заказы за Фев–Май УЖЕ есть в таблице строками (Фев = строки 50–107 и т.д.), пустые только
колонки денег E/F/G. Поэтому Apps Script ➕ «Добавить заказы» НЕ используется — он добавляет
НОВЫЕ строки, а нам надо вписать деньги в СУЩЕСТВУЮЩИЕ.
Claude НЕ может писать в таблицу напрямую (Google Drive коннектор только на чтение).
Рабочая схема оцифровки месяца:
1. Олег фотографирует страницы маминой тетради за месяц.
2. Claude считывает тетрадь + сверяет со строками в таблице (тетрадь — главная по деньгам).
3. Claude выдаёт ГОТОВЫЙ блок из 3 колонок (Цена продажи / Себестоимость / Такси) ровно
   по порядку строк месяца.
4. Олег вставляет блок одним вставлением в первую ячейку E нужного месяца (напр. E50 для февраля).
5. Прибыль (колонка J) считается формулой `=N(E)-N(F)-N(G)-N(H)-N(I)` сама.
   Если в строках месяца формулы в J нет — Claude даёт и её отдельным столбцом для вставки.
6. РЕКЛАМА за месяц: Олег вписывает сумму Google Ads ОДНОЙ строкой в колонку H (Реклама)
   на ПОСЛЕДНЕЙ строке месяца. Тогда чистая месяца в Сводке автоматически = Σ прибыли − реклама.
   Текущая реклама ~7,8 млн донгов/мес.
7. После всех месяцев — Claude строит вкладку «Сводка» (выручка/закуп/чистая/средний чек/маржа/
   рост заказов по месяцам + графики). Сводка уже есть и работает на формулах
   COUNTIF/SUMIF по колонке K (Месяц) — новые месяцы подтягиваются сами.

## МЕТОД: SEO-ЛОНГРИДЫ (рерайт статей) — ЗАФИКСИРОВАНО 12 июня 2026

> Как переделывать статьи блога из шаблонных коротышей в живые SEO-лонгриды.
> Эталон = `blog/rozy-v-nyachange-znachenie-uhod-i-komu-darit-ru.html` (~1470 слов, 9 H2, 8 FAQ).
> Скрипт-конвейер лежит: `outputs/rewrite_final.py` (рабочий пример с process() и 3 статьями).

### Зачем
Старые статьи были ~300-500 слов, «2 мин чтения», шаблонные (ДР и годовщина — почти одинаковый скелет → каннибализация запросов в Google, тонкий контент). Лонгрид = больше H2 (больше длинных хвостов-запросов), уникальный угол на каждую статью (нет дублей), расширенный FAQ с JSON-LD FAQPage (может показаться прямо в выдаче Google).

### Целевая планка
- **Объём:** 1000-1500 слов в теле статьи (между `</h1>` и `</article>`). Эталон 1470.
- **Структура:** 8-10 H2-секций + вступление + 8 вопросов FAQ.
- **Уникальный угол у каждой статьи** (НЕ повторять секции между статьями):
  - Розы → язык цвета/числа/сорта + традиции Вьетнама и Кореи + уход в жаре.
  - День рождения → подбор букета ПОД получателя (жена/мама/подруга/коллега/мужчина) + шары + детский ДР + сезонность.
  - Годовщина → символика годовщин ПО ГОДАМ (ситцевая/серебряная/золотая) + романтическая доставка-сюрприз (к завтраку/в номер/в ресторан).
  - Отель → как работает доставка в отель + большой список отелей Нячанга/Камрани (ловит запросы по названиям отелей) + что написать в открытке.

### Что МЕНЯЕМ в файле (5 точек)
1. `<title>`, `og:title`, JSON-LD `headline` — новый цепкий заголовок с гео и «цены/гид».
2. `<meta name="description">` + `og:description` + JSON-LD `description`.
3. Время чтения (бейдж вида `Повод · 8 мин чтения`).
4. Тело статьи между `</h1>` и `</article>` — целиком новый лонгрид.
5. Видимый FAQ-блок + JSON-LD `FAQPage` `mainEntity` (8 вопросов, синхронно).

### Что НЕ ТРОГАЕМ (сохраняется как есть)
Head-бойлерплейт, header/nav, переключатель языков (флаги), footer, карточки related-товаров (секция «Подходящие букеты» после `</article>`), все скрипты — `MARK-CARD-JS`, IntersectionObserver reveal, GA-конверсия `AW-18183091777`. Все MARK-* комментарии — функциональные якоря, НЕ удалять.

### Технический приём (splice)
Тело режется между `</h1>` и `</article>`; FAQ — отдельной regex-заменой по обёртке `<div class="space-y-3">...`; JSON-LD FAQPage — regex по `"@type":"FAQPage","mainEntity":[...]`. Классы разметки (копировать из эталона): H2 `font-serif text-2xl font-bold mt-10 mb-4` + `style="color:#1a1a1a;"`; абзац `text-stone-600 leading-relaxed mb-4`; вступление `text-stone-600 text-lg leading-relaxed mb-2`; список `space-y-2 mb-4 text-stone-600`. Скрипт `process()` идемпотентен (можно гонять повторно — title/desc/rt заменяются только если старое ещё на месте). НЕ запускать `seo/build_site.py` — затрёт ручные правки.

### Проверка перед деплоем
Скрипт-валидатор: баланс тегов (article/section/ul/h2), все JSON-LD парсятся, видимых FAQ ровно 8 = JSON-LD вопросов, GA и MARK-CARD-JS на месте, related-товары целы.
