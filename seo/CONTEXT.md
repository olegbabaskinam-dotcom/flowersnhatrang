### Что это

качественных страниц за год. Усилия Олега — макс 1 час/день.

### Инфраструктура (НАСТРОЕНО, работает)
- Сайт: статичный HTML, папка `new-site/`, домен flowers-nha-trang.online
- Хостинг: GitHub Pages, репозиторий `olegbabaskinam-dotcom/flowersnhatrang`, ветка `main`
- Языки: RU `index.html`, EN `index-en.html`, KO `index-kr.html` + balloons ×3
- Публикация автоматизирована: `seo/publish.sh` (токен в `seo/.env`, в .gitignore)
- Песочница Claude ДОСТАЁТ до GitHub → Claude может сам коммитить/пушить.
  ВАЖНО: git в примонтированной папке падает на правах (index.lock). Рабочий способ
  пуша из песочницы — клонировать репо в /tmp, скопировать туда новые файлы из
  папки сайта, commit + push токеном. (publish.sh — для запуска на Маке Олега.)

### Готовые инструменты в seo/
- `registry.csv` — 365 тем статей на год (с 2026-06-03), колонки: id,date,type,
  category,keyword_ru,title_ru,slug,photo_query_en,status_ru/en/ko. Статус todo→done.
- `download_photos.py` — качает бесплатные фото с Pexels по photo_query_en,
  webp 1200px + thumb 600px в `img/blog/<slug>.webp`. Ключ в .env. Вариативность
  по id. Запуск на Маке: `python3 download_photos.py N`
- `gen_topics.py` — генератор реестра (если надо пересобрать темы)
- `publish.sh` — git add/commit/push на GitHub Pages
- `SEO-RULES.md` — правила страниц (товар/статья), вкл. ротацию отелей
- `.env` — секреты: PEXELS_KEY, GITHUB_TOKEN, GITHUB_REPO (НЕ коммитить)

## 6. Структура сайта (видение Олега)

### Главная страница (сверху вниз)
1. Hero — как сейчас
2. Отзывы
3. Шары (balloons)
4. Кнопка «Статьи» → открывает БЛОГ: красивая сетка карточек с фото

  и на balloons на всех 3 языках тоже. Это общий элемент сайта.
