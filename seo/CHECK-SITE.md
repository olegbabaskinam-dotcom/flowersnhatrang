## ШАГ 1 — Авто-скрипт (технические ошибки)

Из папки `new-site`:
```bash
python3 seo/check_site.py
```
Скрипт ничего не меняет, только читает. Выдаёт три уровня:
- **❌ ERROR** — обязательно чинить (битая ссылка, пропавшее фото, несимметрия языков, запретная фраза «оплата картой», битый sitemap).
- **⚠️ WARN** — проверить глазами (вертикальное фото статьи, страница не в sitemap, две статьи одной категории подряд, img без alt).
- **ℹ️ INFO** — справка (число карточек, URL в sitemap и т.п.).

Что именно проверяет скрипт:
1. **Симметрия языков** — у каждой статьи и товара есть ru/en/ko; одинаковое число карточек в витринах блога и каталога на всех 3 языках; есть все главные трио (index, balloons, catalog, blog).
2. **Битые ссылки** — каждая локальная `href` ведёт на существующий файл.
3. **Пропавшие фото** — каждый локальный `src` (webp/png/jpg/svg) существует.
4. **Sitemap** — валидный XML (xmllint), каждый `<loc>` → существующий файл, каждая страница попала в sitemap, есть hreflang.
5. **Реестр** — число `done` в registry.csv = числу статей в blog/; ротация категорий (нет двух одинаковых подряд).
6. **Запретные фразы** — «оплата картой / pay by card / 신용카드» нигде нет (терминалов нет!).
7. **Главная** — все товары из products.csv есть в `js/featured.js` (иначе не попадут в ротацию).
8. **Фото статей** — горизонтальные (~1200×760); вертикальные обрежутся в карточке.
9. **alt / lang** — у `<html>` есть lang, у `<img>` есть alt.

Если есть ❌ — сначала чиню их, потом перезапускаю скрипт до чистоты.

## ШАГ 2 — Визуальная проверка (глазами, чего скрипт не видит)
