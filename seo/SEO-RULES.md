# 📐 ПРАВИЛА SEO-СТРАНИЦ (полная спецификация)

> Claude следует этим правилам для КАЖДОЙ страницы. Читать вторым — после CONTEXT.md.
> Два типа страниц: **ТОВАР** (букет / шары / корзина) и **СТАТЬЯ** (блог).
> Оба типа — на 3 языках: RU / EN / KO. Цель года: 1000+ страниц, каждая —
> отдельный «магнит» под конкретный запрос, а не шаблонный клон.

---

## 0. ГЛАВНЫЙ ПРИНЦИП — «не страницы ради страниц»

Google ранжирует **полезность и уникальность**, а не количество. 2000 одинаковых
страниц с подменёнными словами = риск фильтра (Helpful Content / spam). Поэтому:

1. **Одна страница = один поисковый интент.** Прежде чем писать, Claude формулирует:
   кто это ищет, что хочет узнать/купить, на каком языке думает.
2. **Каждый текст пишется заново под этот интент.** Нельзя брать абзац с другой
   страницы и менять 2 слова. Меняется угол подачи, структура, примеры, FAQ.
3. **Ценность раньше продажи.** Сначала отвечаем на вопрос человека, потом мягко
   ведём к заказу.
4. **Только реальные факты** из CONTEXT.md (раздел 4). Ничего не выдумываем —
   если факта нет, спрашиваем Олега, а не сочиняем.
5. **Если страница не несёт нового смысла — её не делаем.** Лучше 1 сильная,
   чем 3 пустых.

## 1. ОБЩИЙ ТЕХНИЧЕСКИЙ SEO (на каждой странице, уже реализовано в генераторе)

| Элемент | Правило |
|---|---|
| `<title>` | ≤ 60 симв., главный ключ в начале, ` \| NhaTrang Flowers` в конце |
| `<meta description>` | 140–160 симв., ключ + выгода + действие. Уникальна для каждой страницы и языка |
| `<h1>` | Ровно один, с главным ключом |
| h2/h3 | Иерархия по смыслу, ключи и синонимы в подзаголовках |
| `canonical` | На саму себя (абсолютный URL) |
| `hreflang` | ru / en / ko + `x-default` (=ru). Связывает 3 версии ОДНОЙ страницы |
| Open Graph | og:title, og:description, **og:image = абсолютный URL** (иначе нет превью при шеринге) |
| Schema JSON-LD | Товар → `Product` + `Offer` + `FAQPage`. Статья → `Article` + (если есть вопросы) `FAQPage` |
| Картинки | webp/avif, `loading="lazy"`, осмысленный `alt` с ключом на языке страницы |
| Цена | Локализована: RU «донгов», EN «VND», KO «동». Никогда не оставлять «донгов» на EN/KO |
| Внутренние ссылки | 3–5 живых ссылок: каталог, блог, смежные товары/статьи |
| CTA | WhatsApp + Telegram везде; для KO-страниц добавлять KakaoTalk |
| Навигация | Языково-корректная: переключатель языка ведёт на ТУ ЖЕ страницу в др. языке |
| Скорость | Без тяжёлых скриптов; стили — в общем styles.css + inline (styles.css = purged Tailwind, новые классы там НЕ работают) |
| URL | Человекочитаемый, латиница-транслит: `/catalog/25-krasnyh-roz-ru.html`, `/blog/<slug>-en.html` |

## 2. АНТИ-ШАБЛОН — правила уникальности (САМОЕ ВАЖНОЕ)

Чтобы 1000 страниц не превратились в один шаблон, каждая страница обязана
отличаться по этим осям:

### 2.1. Уникальный текст
- **0 % копипаста** между страницами. Описания, вступления, FAQ — каждый раз новые.
- Title и description — **никогда не повторяются** дословно в пределах сайта.
- Запрещены «универсальные» абзацы-затычки, которые подходят к любому товару.

### 2.2. Разные углы под разные запросы
Под один товар/тему могут быть похожие ключи — но подача разная:
- *«25 красных роз»* → акцент на романтику, признание.
- *«розы с доставкой в отель»* → акцент на сервис, номер, ресепшен.
- *«цветы на годовщину»* → акцент на повод и эмоцию.

### 2.3. Ротация переменных (генератор уже умеет, держать живым)
- **Отель** — по очереди из списка (раздел 5), не один и тот же.
- **Похожие товары / ссылки** — разные на разных страницах.
- **FAQ** — варьировать формулировки и порядок, не один и тот же набор слово-в-слово.
- **Примеры, цифры, сезоны** — подбирать под тему.

### 2.4. Заголовки-вариации
Не лепить везде «Букет X — доставка в Нячанге». Чередовать модели:
«Доставка X в Нячанге день в день», «X в подарок: …», «X — заказать с доставкой …»,
«Сколько стоит X в Нячанге», и т.п. — под реальную формулировку запроса.

### 2.5. Самопроверка на дубль
Перед публикацией Claude мысленно сравнивает новый текст с соседними страницами
той же категории. Если совпадает структура/фразы → переписать.

## 3. ЯЗЫКИ (нативно, не машинный перевод)

- **RU** — русскоязычные туристы и экспаты. Тепло, подробно, оплата рублями/донгами/USDT.
- **EN** — англоязычные туристы. Просто, про сервис: доставка в отель, оплата картой/USD, same-day.
- **KO** — корейцы. Учитывать корейские поводы и этикет дарения, **KakaoTalk как канал**,
  вежливый тон, цена в 동/USD. Не дословный перевод RU, а текст под корейца.
- Каждый язык — самостоятельный текст под СВОЮ аудиторию и её запросы (ключи ищем
  отдельно по каждому языку, не переводим русский ключ буквально).

---

## 4. КЛЮЧИ И ИНТЕНТ

- 1 главный ключ + 2–3 связанных (LSI/синонимы) на страницу.
- Главный ключ в: `title`, `h1`, первом абзаце, одном `alt`, `description`.
- Связанные ключи — в h2/h3 и теле естественно.
- **Без переспама.** Текст для людей; ключ вплетён, а не натыкан.
- Тип интента определяет тип страницы:
  - коммерческий («купить / заказать / цена / доставка X») → **ТОВАР**;
  - информационный («какие / как / когда / что значит») → **СТАТЬЯ**.

## ТИП 1: СТРАНИЦА ТОВАРА (букет / шары / корзина)

### Что даёт Олег вручную
- Фото (главное + 6–7 доп. → `img/<slug>-2.webp … -8.webp`)
- Название, тип/состав, цена, повод.
Claude пишет: описание, состав, FAQ, мета, alt — всё на 3 языках, уникально.

### Структура (как в генераторе build_site.py — держать и наполнять)
1. Хлебные крошки (Каталог / Название).
2. **Hero:** галерея (главное фото + миниатюры) | колонка: 2 бейджа, H1, краткое
   описание, **блок цены (локализованной)**, кнопки CTA, 3 чипа доверия.
3. **Полоса доверия** (500+ доставлено · 5.0/104 отзыва Google · оплата при получении).
4. **3 преимущества** (свежесть · день в день · доставка в отель/виллу) — с иконками.
5. **Описание** (sec-card): 2–3 предложения, эмоция + ключ + 1 отель из ротации.
6. **Доставка и оплата** (sec-card, список с галочками) — факты из CONTEXT.
7. **FAQ** (3–4 вопроса, варьировать): доставка в отель, день в день, оплата, и 1 под товар.
8. **Блок Google-отзывов** (3 скриншота + ссылка).
9. **Похожие букеты** (3 ссылки, ротация) + кнопка «весь каталог».
10. Schema: `Product` + `Offer` (price, VND, InStock) + `FAQPage`.

### Текст
- **120–250 слов** уникального описания на язык (товар — короче статьи, но не шаблон).
- Описание отвечает: что это, кому/на какой повод, почему у нас (свежесть, день в день, в отель).
- alt каждой фото — разный, по-человечески, с ключом языка.

### Особое для шаров (категория «Шары»)
- Подчёркивать состав набора (S/M/L), комбо с цветами, сроки (до 20:00), фото-зону/праздник.
- Ключи: «гелиевые шары Нячанг», «шары на день рождения», «фотозона шарами» и т.п.

## ТИП 2: СТАТЬЯ (БЛОГ) — основной ежедневный продукт

> Каждый день: 1 статья × 3 языка (RU/EN/KO) из `seo/registry.csv` (план на год).
> Это главный канал органического трафика — качество критично.

### Структура (база — варьировать под категорию, см. ниже)
1. **H1** — заголовок с ключом, под реальный запрос.
2. Обложка-фото (по `photo_query_en`, alt с ключом).
3. **Вступление** (2–3 предложения; ключ — в первом; обозначить пользу читателю).
4. **Тело: 4–6 разделов с h2**, в каждом — суть, примеры, где уместно фото/список.
5. **Практика:** списки, советы, мини-таблицы — то, что реально помогает.
6. **Мягкий мостик к товарам:** 2–3 ссылки на подходящие букеты каталога в контексте.
7. **CTA-блок:** закажи доставку — WhatsApp / Telegram (+ KakaoTalk на KO).
8. **FAQ** (3–4 вопроса по теме статьи) + Schema FAQPage.
9. **1 отель** из ротации в контексте доставки.
10. Schema `Article` (headline, image, datePublished, author=NhaTrang Flowers).

### Текст
- **700–1200 слов** уникального текста на язык.
- Польза в первую очередь, продажа — мягко в конце.
- Внутренние ссылки: 2–3 на товары + 1–2 на смежные статьи.

### 🎯 Анти-шаблон для статей: архетип под КАТЕГОРИЮ registry
Чтобы статьи не были одинаковыми, структуру задаёт категория темы:

- **Повод** (день рождения, годовщина…) → эмоция + «какие цветы и почему» + подборка букетов под повод.
- **Гид** (для туриста…) → пошаговый практический гид: как заказать, сроки, оплата, доставка в отель.
- **Цветок** (розы, лилии…) → про сам цветок: значение, уход, сочетания, когда дарить.
- **Отель** → посадочная «доставка цветов в [отель]»: как довозим, сроки, в номер/ресепшен.
- **Культура** → вьетнамские/корейские традиции дарения, символика, что уместно/нет.
- **Сезон** → сезонность цветов и поводов (Tet, лето, праздники), что актуально сейчас.
- **Связка / Комбо** → «X + Y» (цветы+шары, повод+получатель): закрывает длинные хвостовые запросы.
- **Получатель** (маме, девушке, коллеге…) → что уместно конкретному получателю.
- **Шары** → оформление праздника шарами, наборы, фотозона.

Каждый архетип = свой скелет h2-разделов и свой тон. Не использовать один шаблон на все.

## ЦЕЛЬ И МЕТРИКА УСПЕХА (через год)
- 1000+ страниц, **каждая ранжируется по своему запросу** (не каннибализация).
- Рост органики RU + EN + KO; запросы вида «[товар/повод/цветок/отель] Нячанг».
- Заявки в WhatsApp/Telegram/KakaoTalk с органики.
- Признак провала, которого избегаем: масса страниц, ни одна не в топе, потому что
  они шаблонные и конкурируют сами с собой. Поэтому — уникальность каждой страницы.
