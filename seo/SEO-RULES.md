# 📐 ПРАВИЛА НАПИСАНИЯ SEO-СТРАНИЦ

> Claude следует этим правилам для КАЖДОЙ страницы. Читать после CONTEXT.md.
> Два типа страниц: ТОВАР (букет) и СТАТЬЯ (блог). Оба — на 3 языках (RU/EN/KO).

---

## ОБЩИЕ ПРАВИЛА (для всех страниц)

### Технический SEO (обязательно на каждой странице)
- `<title>` — до 60 символов, ключ в начале, бренд в конце
- `<meta name="description">` — 140-160 символов, с ключом и призывом
- 1 `<h1>` на странице (с главным ключом), дальше h2/h3 по иерархии
- `<link rel="canonical">` на саму себя
- `hreflang` на все 3 языковые версии + x-default
- Open Graph: og:title, og:description, og:image
- Schema.org разметка (JSON-LD): Product — для товара, Article — для статьи
- Все `<img>` с `alt` (с ключом, по-человечески), lazy-load, webp/avif
- Внутренние ссылки: 3-5 на каждой странице (на каталог, блог, смежные)
- Быстрая загрузка: без тяжёлых скриптов, фото оптимизированы
- URL человекочитаемый: `/catalog/roses-25-red-ru.html`, `/blog/flowers-for-birthday-en.html`

### Языки
- НЕ машинный перевод. Каждый язык пишется НАТИВНО под свою аудиторию.
- KO: учитывать корейские поводы и привычки, KakaoTalk как канал
- EN: туристы, отели, оплата картой/долларами
- RU: подробно, тепло, оплата рублями/донгами
- hreflang связывает 3 версии между собой

### Ключевые слова
- 1 главный ключ + 2-3 связанных на страницу
- Ключ в: title, h1, первом абзаце, alt одной фото, description
- БЕЗ переспама — текст для людей в первую очередь

### 🏨 ПРАВИЛО ОХВАТА ПО ОТЕЛЯМ (обязательно)
- В описании КАЖДОЙ страницы (товар и статья) упоминать минимум 1 популярный
  отель Нячанга в контексте доступной доставки.
  Пример: «Доставим букет прямо в номер Vinpearl, Muong Thanh, Sheraton и другие отели Нячанга».
- Отели РОТИРОВАТЬ — не повторять один и тот же на каждой странице,
  брать по очереди из списка ниже, чтобы покрыть все названия.
- Цель: ранжироваться по запросам «доставка цветов в [отель] Нячанг» на 3 языках.
- Раз в N страниц можно делать отдельную посадочную под конкретный отель.

## ТИП 1: СТРАНИЦА ТОВАРА (БУКЕТ)

### Структура (сверху вниз)
1. H1 — название букета + ключ («Букет из 25 красных роз — доставка в Нячанге»)
2. Галерея 7-8 фото (листаются), главное фото первым
3. Цена + кнопки CTA (WhatsApp / Telegram / KakaoTalk)
4. Краткое описание (2-3 предложения, эмоция + ключ)
5. Состав букета (список цветов, размер, упаковка)
6. Условия доставки (сроки, зоны, оплата) — берём из CONTEXT
7. Блок «Похожие букеты» (3-4 ссылки на каталог)
8. FAQ (3-4 вопроса: можно ли в отель, доставка день в день, оплата)
9. Schema Product (name, image, description, offers/price, currency)

### Что даёт Олег вручную
- Фото (7-8 шт)
- Название / тип цветов / цена
Claude оформляет описание, состав, FAQ, мета, 3 языка.

### Длина текста
- 250-400 слов уникального текста на язык (не считая FAQ)

## ТИП 2: СТАТЬЯ (БЛОГ)

### Структура (сверху вниз)
1. H1 — заголовок с ключом («Какие цветы подарить на день рождения во Вьетнаме»)
2. Обложка-фото
3. Вступление (2-3 предложения, ключ в первом)
4. Тело: 4-6 разделов с h2, по 1 фото где уместно
5. Практические советы / списки (читаемо)
6. Мягкий переход к товарам — 2-3 ссылки на подходящие букеты каталога
7. CTA-блок (закажи доставку — WhatsApp/Telegram/KakaoTalk)
8. FAQ (3-4 вопроса по теме статьи)
9. Schema Article (headline, image, author, datePublished)

### Длина текста
- 700-1200 слов уникального текста на язык
- Польза в первую очередь, продажа — мягко в конце

### Темы статей
- Берутся из `seo/registry.csv` (план на год)
- Каждая решает реальный поисковый запрос аудитории
